# GhostClaw System Glossary

## Purpose

This glossary defines the canonical terminology used throughout the GhostClaw ecosystem.

GhostClaw is a complex AI operating system with many interacting components:

- runtime
- planner
- agents
- skills
- marketplace
- autonomous companies

Without consistent terminology the documentation, runtime instructions, and agent coordination can drift.

This glossary provides the **single source of truth** for GhostClaw vocabulary.

# Runtime Concepts

## Signal

Signals are events detected by the system.

Examples:

- keyword_opportunity_detected
- marketplace_gap_detected
- runtime_error_detected
- vertical_demand_spike

Signals trigger planning and execution workflows. :contentReference[oaicite:2]{index=2}

---

## Plan

A plan is the planner's response to a signal.

A plan contains:

- planner action
- required agents
- required skills
- expected outputs
- priority
- escalation path

Plans are generated by the **Master Planner Agent**.

---

## Job

A job is the smallest unit of executable work.

Jobs are created from plans and executed by agents.

Jobs move through states such as:

proposed  
queued  
assigned  
running  
blocked  
completed  
failed  
published :contentReference[oaicite:3]{index=3}

---

## Artifact

An artifact is an output created during execution.

Examples:

- documents
- code
- marketplace listings
- websites
- reports
- company blueprints

Artifacts are stored in the runtime persistence layer.

Ecosystem :contentReference[oaicite:4]{index=4}

## CEO Agent

A CEO Agent is responsible for strategic decisions within a domain.

Examples:

Marketplace CEO Agent

CEO agents define goals that the planner converts into actions.

## Master Planner Agent

The planner responsible for converting signals into plans and tasks.

launch_new_company  
generate_content_cluster

- priority

# Capability Concepts

## Skill

A skill is a reusable capability used by agents.

Examples:

Keyword Research  
SEO Audit  
Content Generation

Skills can be installed through Ghost Mart. :contentReference[oaicite:6]{index=6}

---

## Tool

A tool is the implementation that powers a skill.

Example:

Skill: Backlink Outreach  
Tool: Email outreach automation system

---

## Automation Pack

An automation pack is a bundle of agents and skills designed to execute a multi-step workflow.

Example:

SEO Growth Engine

Automation packs allow complex operations to run automatically.
