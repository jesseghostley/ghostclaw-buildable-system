# GhostClaw Naming, Origin, and Terminology Alignment

## Purpose

This document clarifies the **origin story**, **naming conventions**, and **terminology alignment** of the GhostClaw system.

It was created after the architecture review to ensure:

- consistent naming across the documentation
- a clear explanation of how the system evolved
- alignment between the runtime, planner, agents, marketplace, and ecosystem components

This document does **not change the architecture**.  
It only clarifies the story and terminology.

---

# 1. Origin of GhostClaw

GhostClaw was not created through keyword research or SEO strategy.

The concept emerged organically from studying how companies gain traction and scale.

The original inspiration came from the book:

**Traction – by Gino Wickman**

The book describes how strong organizations operate with:

- clear vision
- defined responsibilities
- repeatable processes
- disciplined execution

GhostClaw applies those same ideas to **AI agents and autonomous systems**.

Instead of people filling roles inside a company, **AI agents fill operational roles inside a digital organization**.

GhostClaw is therefore designed as:

**An AI operating system that organizes agents the same way Traction organizes teams.**

---

# 2. The EOS → AI Workforce Translation

Traction describes several organizational roles:

Visionary

Leadership team

Operational teams

GhostClaw mirrors this structure with agents.

| Traction Role | GhostClaw Equivalent |
|---|---|

# 4. Terminology Alignment

During the architecture review, several naming inconsistencies were identified.

The following standard terminology is now recommended.

---

## Skills vs Tools

Current usage varies between **skills** and **tools**.

Recommended standard:

Skill = atomic capability executed by agents  
Tool = implementation that powers a skill

Example:

Skill: "Backlink Outreach"  
Tool: "Email Outreach Automation Script"

---

## Agents vs Workers

Agents should remain the standard term.

Agent = an AI role responsible for performing tasks.

Examples:

SEO CEO Agent  
Research Agent  
Content Agent  
Marketplace Analyst Agent

---

## Automation Packs

Automation packs are bundles of capabilities.

Automation Pack = bundle of agents + skills + workflows.

Example:

AI SEO Automation Pack.

---

## Blueprints

Blueprints describe how systems or companies are assembled.

Blueprint types:

Platform Blueprint  
Automation Blueprint  
Company Blueprint  

Example:

AI SEO Agency Blueprint.

# 7. Why the System is Called GhostClaw

The name represents two ideas.

Ghost

The system operates autonomously and invisibly in the background.

Claw

The system actively builds, expands, and deploys capabilities.

Together the name reflects a system that:

- quietly operates
- aggressively builds
- continuously expands

---

# 8. Naming Guidelines for Future Components

When creating new system components:

1. Use **Ghost** prefix for core platform components.
2. Use **Agent** for workforce roles.
3. Use **Skill** for capabilities.
4. Use **Blueprint** for structured system designs.
5. Use **Pack** for bundled capabilities.
